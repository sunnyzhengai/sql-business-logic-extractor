What this design demonstrates

  • Entity-first navigation (the ARPB case) -- the SME's immediate question "what touches this table?" gets a direct answer instead of needing to find the right community first.
  • Multi-field semantic search (the asthma case) -- a single search term hits view names, descriptions, comments, AND code-to-meaning lookups simultaneously. The result groups tell the SME WHERE the match came from so they can judge relevance.
  • Epic-specific concept search (the registry case) -- no new infrastructure; falls out of the first two for free, with naming- convention badges layered on top.
  • Code resolution as a force multiplier -- the most visually striking moment is when the asthma search hits DX_ID = 1234 in a SQL filter and surfaces it as "Asthma, unspecified" via the CLARITY_EDG lookup. That's the value-add the rest of the artifact set can't replicate.
