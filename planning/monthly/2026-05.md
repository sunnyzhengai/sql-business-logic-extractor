# May 2026 — Tool Development

## North star
Ship 4 SQL extractor/generator tools, each in two flavors (with-AI and without-AI), ready to wrap with a website in June.

## Themes
1. **Column extractor** — extract table + column inventory from SQL
2. **Technical logic extractor** — joins, filters, transformations, lineage
3. **Business logic extractor** — semantic/business-rule layer on top of technical
4. **Report description generator** — natural-language description of what a SQL report does

## Key deliverables (one tool per week)

- [ ] **Week 1 (May 4-10):** Column extractor — non-AI only
- [ ] **Week 2 (May 11-17):** Technical logic extractor — non-AI only
- [ ] **Week 3 (May 18-24):** Business logic extractor — both AI and non-AI
- [ ] **Week 4 (May 25-31):** Report description generator — both AI and non-AI

## Definition of done (per tool)
- CLI entrypoint that runs end-to-end on a sample SQL file
- Test fixture in `tests/` covering golden-path output
- Both versions produce comparable outputs (so AI vs non-AI is benchmarkable)
- README snippet showing usage

## Risks / unknowns
- Business logic extractor is the fuzziest scope — define what "business logic" means concretely before building
- AI versions: pick one model family and stick with it for May (avoid swap-cost)
- Existing code in `sql_logic_extractor/` already covers parts of #1 and #2 — inventory before duplicating
