## REFERRAL_ID (passthrough)
   Domain: Referral Tracking

   ### Technical Definition (for SQL developers)

   Expression: REFERRAL.REFERRAL_ID

   Base Tables: REFERRAL
   Base Columns: REFERRAL.REFERRAL_ID
   Filters:
     - ra.appt_sequence_for_referral = 1 OR ra.appt_sequence_for_referral IS NULL
     - COUNT(DISTINCT REFERRAL_ID) >= 10
     - r.ENTRY_DATE >= DATEADD(YEAR, -2, GETDATE())
     - NOT r.REFERRAL_STATUS_C IS NULL
   Transformation Chain:
     -> query.REFERRAL_ID (passthrough)
     -> referral_appointments.REFERRAL_ID (passthrough)
     -> base_referrals.REFERRAL_ID (passthrough)

   ### Business Definition (for business users)

   Unique identifier for referrals that are either the first appointment in a sequence or have no sequence, with at least 10 distinct referrals, entered in the last 2 years, and have a non-null status.

## age_category (case)
   Domain: Patient Demographics

   ### Technical Definition (for SQL developers)

   Expression: CASE WHEN DATEDIFF(YEAR, p.BIRTH_DATE, GETDATE()) < 18 THEN 'Pediatric' WHEN DATEDIFF(YEAR, p.BIRTH_DATE, GETDATE()) BETWEEN 18 AND 40 THEN 'Young Adult' WHEN DATEDIFF(YEAR, p.BIRTH_DATE, GETDATE()) BETWEEN 41 AND 65 THEN 'Middle Age' ELSE 'Senior' END /* Age-based risk category */

   Base Tables: PATIENT
   Base Columns: PATIENT.YEAR, PATIENT.BIRTH_DATE
   Filters:
     - ra.appt_sequence_for_referral = 1 OR ra.appt_sequence_for_referral IS NULL
     - COUNT(DISTINCT REFERRAL_ID) >= 10
     - pl.PAT_ID = p.PAT_ID
     - pl.RESOLVED_DATE IS NULL
   Transformation Chain:
     -> query.age_category (passthrough)
     -> referral_appointments.age_category (passthrough)
     -> patient_risk.age_category (case)

   ### Business Definition (for business users)

   Categorizes age as 'Pediatric' if under 18, 'Young Adult' if between 18 and 40, 'Middle Age' if between 41 and 65, and 'Senior' if over 65, based on the difference in years between the current date and the birth date.

   Base Tables: F_SCHED_APPT, pl, PATIENT, REFERRAL
   Base Columns: F_SCHED_APPT.PAT_ID, F_SCHED_APPT.APPT_DTTM, F_SCHED_APPT.APPT_STATUS_C, pl.PAT_ID, PATIENT.PAT_ID, pl.RESOLVED_DATE, .DAY, REFERRAL.ENTRY_DATE, PATIENT.YEAR, PATIENT.BIRTH_DATE
   Filters:
     - ra.appt_sequence_for_referral = 1 OR ra.appt_sequence_for_referral IS NULL
     - COUNT(DISTINCT REFERRAL_ID) >= 10
     - fa.APPT_DTTM >= DATEADD(YEAR, -2, GETDATE())
     - pl.PAT_ID = p.PAT_ID
     - pl.RESOLVED_DATE IS NULL
     - r.ENTRY_DATE >= DATEADD(YEAR, -2, GETDATE())
     - NOT r.REFERRAL_STATUS_C IS NULL
     - appt_sequence_for_referral = 1 OR appt_sequence_for_referral IS NULL
   Transformation Chain:
     -> query.patient_risk_flag (case)
     -> referral_appointments.cumulative_no_shows (passthrough)
     -> appointment_history.cumulative_no_shows (window)
     -> referral_appointments.active_problem_count (passthrough)
     -> patient_risk.active_problem_count (subquery)
